Magasság - Távolság konverzió

Visual Studio Code-ban

Vezérlés működésének bemutatása

A program általános működése

A program működése egy menürendszeren alapul, amely lehetővé teszi, hogy a felhasználó kiválassza a kívánt átváltást. A program indulásakor egy végtelen ciklus (while True) indul el, amely addig fut, amíg a felhasználó ki nem lép. A felhasználó először kiválasztja, hogy magasságot vagy távolságot szeretne átváltani, majd megadja az átváltás irányát, vagyis hogy metrikus rendszerből angolszászba vagy fordítva szeretne váltani. Ezután a program bekéri a szükséges adatokat, elvégzi a számítást, és kiírja az eredményt.

A függvények szerepe a programban

A program elején több külön függvény található, amelyek mind egy-egy konkrét átváltási feladatot látnak el. Például a cmfoot() függvény a centimétert alakítja át lábbá, míg a cminch() a centimétert hüvelykké konvertálja. A távolság átváltására szolgáló függvények, mint például a milekm() és a kmmile(), a mérföld és kilométer közötti váltást végzik el. A függvények használata azért előnyös, mert így a kód átláthatóbb, könnyebben karbantartható, és az egyes műveletek külön-külön is újra felhasználhatók.

A változók működése és szerepe

A programban több változó is szerepel, amelyek a felhasználó által megadott adatokat és a program állapotát tárolják. A fomenu változó a főmenü választását tárolja, vagyis hogy a felhasználó magasságot, távolságot vagy kilépést választott. Az almenu változó az átváltás irányát jelöli, tehát azt, hogy metrikus vagy angolszász rendszerből történjen az átváltás. A cmnagassag, footmag és inchmag változók a magasság megadására szolgálnak különböző formátumokban. A távolság esetében a meter tav, yardtav, kilometer és merfold változók tárolják a felhasználó által megadott értékeket. A tipus változó azt határozza meg, hogy méter–yard vagy kilométer–mérföld átváltás történjen.

A magasság átváltás folyamata

A magasság átváltása két irányban történhet. Ha a felhasználó centiméterből szeretne átváltani, akkor a program először kiszámolja, hogy hány teljes láb fér bele az adott értékbe, majd a maradék centimétert hüvelykké alakítja. Ez úgy történik, hogy a centiméter értéket elosztja 30,48-cal, ami egy láb hossza, majd a maradékot elosztja 2,54-gyel, ami egy hüvelyknek felel meg. Ha az átváltás fordított irányban történik, tehát láb és hüvelyk van megadva, akkor a program külön-külön átszámítja ezeket centiméterre, majd összeadja az eredményeket.

A távolság átváltás működése

A távolság átváltása szintén két fő típusra bontható. Az egyik esetben a program méter és yard között vált, a másik esetben kilométer és mérföld között. A méter–yard átváltásnál a program a métert elosztja 0,9144-gyel, míg a yardot megszorozza ezzel az értékkel. A kilométer–mérföld átváltásnál a kilométert elosztja 1,60934-gyel, a mérföldet pedig megszorozza ezzel az értékkel.

A program vezérlési szerkezete

A program működésének alapját a vezérlési szerkezetek adják. A while True ciklus biztosítja, hogy a program folyamatosan fusson, és többször is lehessen használni újraindítás nélkül. Az if és elif elágazások segítségével a program eldönti, hogy a felhasználó milyen műveletet szeretne végrehajtani, és ennek megfelelően melyik kódrészlet fusson le. A break utasítás lehetővé teszi a programból való kilépést. Ezek az elemek együtt biztosítják a program logikus és jól követhető működését.

Példa a gyakorlati alkalmazásra

Amikor elindítom a programot, először megjelenik egy kérdés, amelyben ki kell választanom, hogy milyen átváltást szeretnék végezni.

Itt én a „T” betűt írom be, mert távolságot szeretnék átváltani. Ezzel a program a távolság átváltás részhez lép, ahol további kérdéseket kapok a pontos művelet meghatározásához.

Méter átváltása yardra

AA következő lépésben meg kell adnom, hogy milyen típusú távolságot szeretnék átváltani. Itt az „M” betűt választom, mert métert szeretnék yardra átváltani. Ezután a program megkérdezi az átváltás irányát is, ahol ismét „M”-et írok be, mivel metrikus rendszerből szeretnék angolszász rendszerbe váltani. Ezután megadom a távolságot, ami ebben az esetben 70 méter. A program kiszámolja az eredményt, és kiírja, hogy ez hány yardnak felel meg.

Újabb művelet indítása

Miután az átváltás megtörtént, a program nem áll le, hanem újra megjeleníti az első kérdést. Ez számomra azért hasznos, mert így több különböző átváltást is el tudok végezni egymás után anélkül, hogy újra kellene indítanom a programot.

Magasság átváltása angolszászból

A következő lépésben a főmenüben az „M” betűt választom, mert most magasságot szeretnék átváltani. Ezután az „A” opciót adom meg, mivel angolszász mértékegységből szeretnék metrikusba váltani. A program ezután bekéri az adatokat: először megadom a láb értéket, ami 19, majd a hüvelyk értéket, ami 4. A program ezek alapján kiszámolja a teljes magasságot centiméterben, és megjeleníti az eredményt.

Kilépés a programból

Az átváltás után a program ismét visszatér a főmenühöz, és újra megjeleníti a kérdést. Ekkor már nem szeretnék további műveleteket végezni, ezért a „K” betűt írom be. Ennek hatására a program kilép, és befejeződik a futás.

A projekt célja

A projekt célja egy olyan Python program elkészítése volt, amely képes különböző mértékegységek közötti átváltásra. A program elsősorban a magasság és a távolság átváltására fókuszál, és lehetőséget biztosít a metrikus (például centiméter, méter, kilométer) és az angolszász (például láb, hüvelyk, yard, mérföld) mértékegységek közötti konverzióra.

A program célja nemcsak a gyakorlati használhatóság, hanem az is, hogy bemutassa a Python programozási alapjait, például a függvények, ciklusok és elágazások használatát.

Mit tanultam belőle?

A projekt elkészítése során jobban megértettem a Python alapú programozás működését, különösen a függvények, ciklusok és elágazások használatát. Megtanultam, hogyan lehet egy programot logikusan felépíteni úgy, hogy a felhasználó különböző lehetőségek közül választhasson egy menürendszeren keresztül.

Gyakorlatot szereztem abban, hogy hogyan lehet matematikai műveleteket programkódban alkalmazni. Emellett fejlődött a problémamegoldó gondolkodásom is, mivel meg kellett terveznem a program működésének lépéseit és a felhasználói bemenetek kezelését.
